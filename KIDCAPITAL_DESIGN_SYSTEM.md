# KidCapital — Design System v2.0
# "Penny's World" — Monopoly Go Inspired

> **This file constrains all visual decisions in KidCapital.**
> Every component, screen, and animation MUST follow these rules.
> When in doubt, reference Monopoly Go, Coin Master, and Board Kings — NOT Tailwind UI, shadcn, or any "dashboard" aesthetic.

---

## 1. IDENTITY

KidCapital is a **mobile board game**, not an app. It must LOOK and FEEL like a game.
The mascot is **Penny the Pig** — a cartoon pig who teaches kids about money.

### What KidCapital IS:
- A colorful, energetic, rewarding board game
- Shiny, tactile, toylike — things look like you can touch them
- Maximalist where it matters (celebrations, rewards, board) and clean where it matters (modals, text, stats)

### What KidCapital is NOT:
- A fintech dashboard
- A SaaS product
- A "dark mode UI" with glassmorphism
- Anything that looks like it was generated by ChatGPT

---

## 2. ANTI-PATTERNS — BANNED ELEMENTS

These are the #1 tells of AI-generated game UI. **NEVER use any of these:**

| Banned | Why | Use Instead |
|--------|-----|-------------|
| `linear-gradient(160deg, #1a1a2e, #16213e, #0f3460)` | The default AI navy gradient | Solid background colors or illustrated backgrounds |
| `backdrop-filter: blur()` / `glass` / `glass-dark` | Glassmorphism = AI slop | Solid surfaces with subtle borders |
| `shadow-glow-gold` / `box-shadow: 0 0 Xpx rgba(245,158,11)` | Neon glow = AI aesthetic | Solid drop shadows for depth |
| `bg-gradient-to-r from-amber-400 to-amber-500` on every button | Every AI app does this | Per-context button colors (see Buttons section) |
| `text-transparent bg-clip-text bg-gradient-to-r` | Gradient text = instant AI flag | Solid colored text |
| `animate-pulse` on everything | Lazy attention-grabbing | Use only on genuinely urgent elements (debt warning) |
| `rgba(255,255,255,0.05)` / `bg-white/5` cards on dark bg | The AI "subtle card" trick | Solid colored cards with real borders |
| Tailwind color names as design tokens (amber-400, emerald-500) | Generic palette | Custom brand colors (see Colors section) |
| `rounded-full` on everything | Pill shapes everywhere | Mix of border-radius values |
| `font-display` on every text element | Over-applying display font | Display font for headers ONLY |

4. **Gold is for rewards ONLY** — coins, payday, celebrations. Not for generic buttons.
5. **No color with opacity below 0.3** — if you can barely see it, it shouldn't exist.

## 4. TYPOGRAPHY

### Font Stack
```css
/* Display / Headers — fun, bold, game-like */
font-family: 'Baloo 2', 'Fredoka', cursive, sans-serif;

/* Body / UI — clean readability */
font-family: 'Nunito', 'Quicksand', system-ui, sans-serif;
```

### Rules:
- **Display font ('Baloo 2')**: Headers, player names, money amounts, button labels. NEVER for body text or descriptions.
- **Body font ('Nunito')**: Descriptions, explanations, Penny's dialogue, modal body text.
- **Minimum font size**: 12px. Nothing smaller. Kids need to read this.
- **Money amounts are ALWAYS display font, ALWAYS bold, ALWAYS colored** (green for positive, red for negative).
- **No gradient text. No text shadows. No outlined text.**

## 5. COMPONENTS

### 5.1 Buttons

Buttons in KidCapital look **3D and pressable** — like physical game buttons.

```css
/* Primary action (Roll!, Collect!, Start) */
.btn-primary {
  background: var(--kc-gold);
  color: #4A3800;
  border: none;
  border-radius: 16px;
  padding: 14px 28px;
  font-family: 'Baloo 2', cursive;
  font-weight: 700;
  font-size: 18px;
  box-shadow: 0 5px 0 var(--kc-gold-shadow), 0 8px 16px rgba(0,0,0,0.2);
  transform: translateY(0);
  transition: transform 0.1s, box-shadow 0.1s;
}
.btn-primary:active {
  transform: translateY(4px);
  box-shadow: 0 1px 0 var(--kc-gold-shadow), 0 2px 4px rgba(0,0,0,0.2);
}

/* Secondary (Skip, Close, Next) */
.btn-secondary {
  background: white;
  color: var(--kc-text-primary);
  border: 2.5px solid #E0E0E0;
  border-radius: 14px;
  box-shadow: 0 3px 0 #D0D0D0;
}

/* Danger (Buy temptation) */
.btn-danger {
  background: var(--kc-temptation);
  color: white;
  box-shadow: 0 5px 0 var(--kc-temptation-dark);
}

/* Success (Invest, Save) */
.btn-success {
  background: var(--kc-invest);
  color: white;
  box-shadow: 0 5px 0 var(--kc-invest-dark);
}
```

**Key principle**: Every button has a `box-shadow` bottom border that makes it look raised. On `:active`, it `translateY(4px)` and the shadow shrinks — like pressing a physical button.

### 5.2 Board Spaces

Board spaces look like **3D tiles/chips on a felt table**.

```
Structure:
┌─────────────┐
│  ┌───────┐  │  ← Outer: subtle shadow for depth
│  │  🏪   │  │  ← Inner: colored tile with rounded corners
│  │ Invest │  │  ← Label below icon
│  └───────┘  │
│   🦊        │  ← Player tokens below
└─────────────┘
```

- Each space type has its OWN background color (not white/5 with a colored border)
- Active space: slightly larger (scale 1.08), stronger shadow, subtle bounce animation
- Inactive spaces: slightly desaturated, smaller
- Space tiles have a **2-3px solid darker border** of the same color family (not white/10)
- Player tokens sit BELOW the space, not overlapping

### 5.3 Cards & Modals (Bottom Sheets)

Modals are **white cards** that slide up. They look like physical game cards.

- Background: `#FFFFFF` — solid white, NO transparency
- Border-radius: `24px` top corners
- NO backdrop-blur. Use a solid `rgba(0,0,0,0.5)` overlay instead.
- Top accent: a **thick (4px) colored bar** matching the space type color
- Content uses the body font (Nunito), not display font everywhere
- Buttons at the bottom follow the 3D button style above
- Card sections use light tinted backgrounds (`#FFF8E1` for gold sections, `#E8F5E9` for green sections) — NOT `rgba(color, 0.05)`

### 5.4 Player Row / HUD

The player HUD sits on the game board — it needs to be **readable but not dominant**.

- Player cards: solid colored backgrounds per player, NOT glass/transparent
- Active player card: thicker border (3px), slightly elevated shadow, a small animated indicator
- Stats (cash, income) use the display font, colored
- Progress ring around avatar: thicker (3px stroke), solid color, not translucent
- No `bg-white/5`, `bg-white/10`, `bg-white/15` — use real colors

### 5.5 Penny Bubble

Penny's speech bubble should look like a **comic book speech bubble**.

- White background, solid
- A small **triangle/arrow** pointing to Penny's avatar (CSS triangle or SVG)
- Rounded corners but NOT `rounded-full`
- Penny's avatar: displayed as an illustrated image (AI-generated asset), NOT just the 🐷 emoji
- Text in Nunito (body font), conversational size (14-15px)

### 5.6 Coin Animation

When gaining/losing money, show **actual coin visuals**.

- Use AI-generated gold coin asset or CSS-drawn coin (circle with $ symbol)
- Coins should **fly/rain** from top, not just fade in as text
- Positive: gold coins rain down with a satisfying arc
- Negative: coins fly AWAY from the player, fading out
- The amount text appears AFTER the animation, not during

### 5.7 Dice

The dice should look like a **physical 3D die**.

- White face with rounded corners
- Subtle 3D perspective (slight rotation in CSS)
- Dots are solid dark circles, properly positioned
- Rolling animation: actual rotation (rotateX, rotateY), not just scaling
- Result: die settles with a small bounce, then shows the face
- Shadow beneath the die changes during roll

## 7. SCREENS

### 7.1 Splash Screen
- Penny (illustrated) center stage, large, animated (bouncing/waving)

- Background: the game-table green with a subtle radial light in the center

## 9. ANIMATIONS

### Principles:
- **Spring physics** for movement (framer-motion spring, not linear/ease)
- **Overshoot and settle** for things landing (dice, tokens moving)
- **Scale bounce** for rewards (coin collect, payday)
- **NO infinite pulse** except for genuinely urgent warnings (debt)
- **NO glow animations** — use scale/position/rotation instead

### Specific Animations:
| Element | Animation | Timing |
|---------|-----------|--------|
| Dice roll | rotateX + rotateY + slight bounce settle | 800ms roll, 200ms settle |
| Token move | Position slide along board path with arc | 300ms per space |
| Coin collect | Coins fly in arc from source to cash display | 600ms |

## 10. DARK MODE

KidCapital does **NOT have a dark mode**. 

The game board area uses the green felt background, which provides the "dark" contrast. Modals and cards are always white/light. This is intentional — kids games don't have dark modes, and it prevents falling back into the AI navy-blue-gradient trap.

If iOS system dark mode is detected, we can slightly darken the sky area and game board, but modals/cards stay light.
